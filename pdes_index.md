## System Pipeline How It Works: Input → Process → Output
PDES follows a strictly organized directory architecture:

- `input/` — **USER:** Drop your raw files and data here for scanning.
- `skills/` — **ENGINE:** Core logic files that process your inputs. (source — don't modify)
- `libraries/` — **DATA:** Read-only registries, schemas, and knowledge bases. (source — don't modify)
- `templates/` — **SOPs:** Pre-defined formats for execution.
- `output/` — **ARTIFACTS:** All generated reports, logs, and trackers.

### 1. Core Skills (The 6-Phase Engine)
The universal workflow applied to every domain you want to master.

| Phase | Skill | Command | Purpose |
| :--- | :--- | :--- | :--- |
| **01** | `core_perceive` | `/perceive` | Assess, diagnose, and map the current state. |

| **03** | `core_design` | `/design` | Create actionable frameworks and protocols. |

### 2. Level Skills (The 32-Level Ladder)
Mapping CS concepts (from BIOS to Quantum) to human development stages and life challenge. `[level_id] = G{z|0≤z≤30}`
- *Check `pdes_index.csv` for the full 32-level listing.*
- Use command `/level [level_id]` to load the appropriate level skill.

### 3. Plugin Skills (Niche Expansions)
Specialized context for specific industries:
- `/niche [area]` — Niche-specific expansion.
- `/niche trading` — Investment Operating System.
- `/niche content` — Repurposing & Lead Generation.
- `/niche health` — Biohacking & Body System.

## Quick Start
1. Run `/pdes` to initialize the system.
2. (Optional) Drop any context files into `input/` for the system to scan
3. Run `/perceive [area]` to scan your input and diagnose your current level.
4. Follow the system's recommendation to load the appropriate `/level g[x]`.
5. Move through the 6-Phase pipeline.
6. Load a plugin if working in a specific niche: `/niche [name]`
7. Check your `output/` folder for implementation packages and trackers.
